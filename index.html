OBJECTS - IN JAVASCRIPT

  • OBJECTS CAN BE DEFINED AS A COLLECTION OF NAMED VALUES;

  • JAVASCRIPT VARIABLES CAN CONTAIN SINGLE VALUES OR MANY VALUES;
    • BECAUSE OF THIS, OBJECTS ARE CONSIDERED VARIABLES.

  • ALMOST EVERYTHING IS AN OBJECT INSIDE OF JAVASCRIPT:
    • VALUES THAT ARE ALWAYS OBJECTS:
      • DATES;
      • MATHS;
      • REGULAR EXPRESSIONS;
      • ARRAYS;
      • FUNCTIONS;
      • OBJECTS.

    • VALUES THAT ARE OBJECTS WHEN DEFINED WITH THE NEW KEYWORD:
      • BOOLEANS;
      • NUMBERS;
      • STRINGS.

    • VALUES THAT AREN'T OBJECTS: (THE ONLY EXCEPTION)
      • PRIMITIVES.

    • Primitive Values

    • Primitive values can be simply defined as data that is not an object and has no methods ( primitive value and primitive data type )
    • Primitive values:
      • Don't have PROPERTIES;
      • Don't have METHODS;
      • Are immutable.
    • Primitive Data Type
      • Primitive Data Type is just data that has a primitive value.
    • Primitive data types:
      • string;
      • number;
      • bigint;
      • boolean;
      • null;
      • undefined;
      • symbol.

    • Properties are the most important part of any JavaScript object.

      • Primitive values

    Deleting properties

    • To delete properties from a object you use the "delete" keyword;
      • The delete keyword erases both the value of the property and the property itself;
      • The delete keyword does not delete inherited properties;
      • Deleting prototype properties will affect all objects inherited from the prototype.
    • The delete operator is designed to be used on "object properties". It has NO EFFECT on variables or functions;
    • The delete operator should not be used on properties from objects that are predefined by JavaScript. It can crash your application.
    • Ex: Deleting the properties nationality and age
    							
const person = {
	firstName: "John",
	lastName: "Doe",
	age:25,
	nationality: "German"
}; 
console.log(person.nationality); //German
console.log(person.age); //25


delete person.nationality;
console.log(person.nationality); //undefined

delete person["age"];  //Different notation
console.log(person.age) //undefined

    Nested Objects

    • Objects can be nested inside other Objects;
    • Nested objects can be accessed using the Dot Notation or Bracket Notation;
    • Ex: Creating a simple nested object called 'cars'
    							
myObj = {
	name:"John",
	age:30,
	cars: {
		car1: "Porsche",
		car2: "Mercedes",
		car3: "Ferrari"
	}
}

console.log( myObj.cars.car1 );	//Porsche
console.log( myObj.cars["car2"] );	//Mercedes
console.log( myObj.["cars"]["car3"] );	//Ferrari

    Creating Objects

    Objects can be created in 4 different ways:

    • Using an object literal:
      • This is the easiest way to create an Object.
      • An object literal is a list of "name:value" pairs (like age:50) inside curly braces {}.
      • Using an object literal, you both define and create an object in one statement.
      • Example: Creating a JavaScript object with four properties
        • 								
const person = { firstName:"John", lastName:"Doe", age:50, eyeColor:"blue" };
      • Example: Creating an empty JavaScript object and then adding four properties
        • 								
const person = {};
person.firstName = "John";
person.lastName = "Doe";
person.age = 50;
person.eyeColor = "blue";
    • Using the "new" keyword
      • For readability, simplicity and execution speed, the object literal method is preferred.
      • Example: Creating a new JavaScript object using "new Object()" and adding 4 properties
        • 								
const person = new Object();
person.firstName = "John";
person.lastName = "Doe";
person.age = 50;
person.eyeColor = "blue";

    Object Accessors ( Getters and Setters )e

    • ECMAScript 5 introduced Getters and Setters
    • Getters and Setters allow you to define Object Accessors (Computed Properties)

    Get

    • With the use of "get" we can return properties from the object.
    • Example: Defining a property called "fullName" and using "get" to return other properties from the "person" object.
      • 								
const person = {
firstName: "John",
lastName: "Doe",
get fullName(){
	return `${this.firstName} ${this.lastName}`;
}
}
console.log( person.fullName ) //output: John Doe

    Set

    • With the use of "set" we can set the values of properties inside the Object
    • Example: Defining a property called "myLastName" and with the use of "set" we can set the values of properties inside the "person" object.
      • 								
const person = {
	firstName: "John",
	lastName: "",
	set myLastName(value){
		this.lastName = value;
	}
}
person.myLastName = "Doe";
console.log( person.lastName ); //Doe

    Benefits of "Getters and Setters" over "Functions"

    • Getters and Setters are better than just using "Functions":
      • JavaScript can secure better data quality when using getters and setters;
      • It gives simpler syntax;
      • Allows equal syntax for properties and methods;
      • Can secure better data quality;
      • Useful for doing things behind-the-scenes.

    Object Constructors

    • As the name already entails, Object Constructors are used for one things, constructing Objects
    • In better words, an Object Contructors is a FUNCTION that is going to be used as a blueprint to make Objects
    • Object Contructors can be used to create an unlimited amount of Objects, that are going to have the same properties and methods between themselves
    • The keyword "this" inside Object Constructors
      • The keyword "this" in an Object Constructor will remain blank, until a new Object is created, and, right after being created, the keyword "this" will refer to the Object that was created itself
      • Everytime you create a new Object with the Object Constructor Function, the keyword "this" will refer to the newly created object, in this way, Object Constructors can have the keyword "this" inside of them with no worries, as the keyword will work just fine for each object being created
    • Creating an Object Contructor: 
      							
function Person(first, last, age) {
	this.firstName = first;
	this.lastName = last;
	this.age 	      = age;
	this.fullName = function() {
		return `${this.firstName} ${this.lastName}`;
	};
}

    • PROTOTYPE PROPERTY
      • To add Properties or Methods to Object Constructors you have to use the "PROTOTYPE" property
      • You can't add properties nor methods to an Object Constructor in the same way you do with normal Objects
      • All Objects in JavaScript inherit properties and methods from a prototype, for example:
        • Date objects inherit from Date.prototype
        • Array objects inherit from Array.prototype
        • Person objects inherit from Person.prototype
      • The "Object.prototype" is in the top of the prototype inheritance chain
      • ALL OBJECTS inherit from the Object.prototype
      • Example:
        Using "prototype" to add a new property called "nationality" 
        									
function Person(first, last, age) {
	this.firstName = first;
	this.lastName = last;
	this.age = age;
}
Person.prototype.nationality = "English";
      • Example:
        Using "prototype" to add a new method called "fullName" 
        									
function Person(first, last, age) {
	this.firstName = first;
	this.lastName = last;
	this.age = age;
}
Person.prototype.name = function() {
	return `${this.firstName} ${this.lastName}`;
